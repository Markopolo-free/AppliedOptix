# Code Change Checklist

- [ ] Summarize the planned change before editing code
- [ ] Check for and respect existing patterns, comments, and previous fixes
- [ ] Update CHANGELOG.md with every significant change
- [ ] Commit to Git after each successful change
- [ ] Add or update tests for critical/logic changes
- [ ] Add code comments for non-obvious logic or important constraints
- [ ] Never remove/overwrite working code without clear user approval
- [ ] Offer to back up data/code before major changes
- [ ] Restart dev server after making a change (always)
